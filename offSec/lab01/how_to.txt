# How to find the RIP offset.
1. generate a cyclic pattern, using the cyclic function in python
    - python3 -c 'from pwn import *; print(cyclic(200).decode())'
2. pass the string in input to the program in GDB. this, will crash the program execution. inside the 
   RET address, we will find a 8bytes (64bits) hex number, which will guide us to find the correct offset of the RIP 
3. you need to decode what you have found. in particular the last 4 bytes (because it is little endian). 
    - to do this, use python3 -c 'from pwn import *; print(cyclic_find(<value_from_crash>))'
   this will tell you the exact offset of the RIP, where you have to put the address of the instruction you want to execute.

# escape room

here, we need to do two things:
1. first, we need to tell the program to execute win function, as done before (find the RIP, use the offset, ...)
2. win function expect two specific parameters. to unlock the shell execution, we need to provide them
3. passing parameters using assembly can be done using ROP gadgets. in particular, we need the "pop rdi; ret" gadget. this, is used to tell the program: pop an address from the stack and 
   put it inside rdi. RDI is used to pass the first parameter to a function. we need to repeat this with "pop rsi; ret" (rsi is for the second parameter)
